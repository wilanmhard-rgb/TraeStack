# TraeStack Office Hours（リポジトリ構成 / 能力境界 / 実行依存）

## 目的

このリポジトリを TraeStack 専用の skills 適配レイヤーとして収束させ、次の内容のみに限定します。

- 適配レイヤー改修：命名、ディレクトリ、ルーティング、ドキュメント、一貫性検証
- 能力収束：Trae プラットフォーム側の責務と、このリポジトリが提供する skill 定義の責務を明確化
- 境界固定：基盤の実行ロジックや既存ワークフローは変更せず、skill の包装と検出性のみを調整

## 現在のリポジトリ構成

- `README.md`：リポジトリ全体の概要
- `skills/`：Trae で検出可能な skill 定義群
  - `skills/<skill-name>/SKILL.md`：各 skill の宣言と利用制約
  - `skills/README.md`：skill 一覧表
  - `skills/ROUTING.md`：シナリオ別の推奨ルーティング
  - `skills/NAVIGATION.zh-CN.md`：中国語ナビゲーション
  - `skills/DELIVERY-SUMMARY.md`：納品サマリー

## 能力境界（このリポジトリが担うもの / 担わないもの）

### このリポジトリが担うもの

- skill 定義レイヤー：各 `SKILL.md` にある `name`、`description`、利用条件、ルール、ワークフロー、出力形式
- ルーティングと情報アーキテクチャ：`ROUTING.md`、`NAVIGATION.zh-CN.md`、`skills/README.md` の整合性
- 適配と収束：命名、パス表現、用語の統一、および残留チェックと一貫性検証

### このリポジトリが担わないもの（今回も変更しない）

- 基盤のコア実行ロジック：実際のツール実行、ブラウザ駆動、ネットワーク取得、デプロイ、アカウント統合などの実行時能力
- 既存ワークフローの実装：skill 内に記載されたフローはガイドであり、このリポジトリ内で実装されるものではない

## 実行依存（安定した呼び出しに必要な外部条件）

- Trae IDE / Trae Agent Runtime：skill の検出、ルーティング、ツール編成を担当
- Trae 内蔵ツール連携：ブラウザ機能、ターミナル実行、診断、Supabase、Stripe など。内容は各 skill の記述に依存
- このリポジトリが保証すべきことは `skills/<name>/SKILL.md` の形式と命名が正しいことのみであり、ディレクトリ名と skill 名の不一致による検出失敗や誤ルーティングを防ぐ

## リスクと制約

- 全面名称変更は過去の呼び出し習慣に影響する可能性があるため、能力を損なわずに外部入口を `traestack` 命名体系へ統一する必要がある
- ドキュメントとルーティングは常に同期されている必要があり、rename 時は `SKILL.md` の frontmatter、ディレクトリ名、索引、ナビゲーションを同時に更新する必要がある

## 今回の納品方針

- すべての外部入口命名を `traestack` に統一する
- 変更対象は skill 包装レイヤー、ドキュメント、ルーティング、検証スクリプトのみに限定し、基盤の実行ロジックには変更を加えない
